# 色彩 — 数学的分解と結論

## 1. 色空間の選択

**提示**: 色はHEX/RGBで指定すればよい。
**反証**: RGB/HSLは知覚的に不均等。HSLの`lightness 50%`は黄色では眩しく青では暗い。同じ「明度差20」でも色相によって見え方が変わり、パレットの一貫性が崩れる。
**結論**: 設計はOKLCHで行い、出力時にHEXへ変換する。
- L（明度 0-1）: 知覚的に均等。UIの階調設計はLの等差数列で組む
- C（彩度）: 0.02以下=ほぼ無彩色、0.15以上=ビビッド
- H（色相角）: 調和関係は角度演算で定義できる

```
背景階調の例（ダークテーマ）:
L = 0.15, 0.20, 0.25, 0.32 （等差 +0.05〜0.07）
テキスト階調: L = 0.95(主), 0.75(副), 0.55(無効)
```

## 2. 配色比率 — 60-30-10

**提示**: 60%ベース / 30%セカンダリ / 10%アクセント。
**反証**: 「cool系」ギャラリー掲載サイトの実測傾向は 80-90% がベース（黒・白・ネイビー）で、アクセントは 5% 以下。60-30-10は雑誌・インテリア由来で、Webの「洗練」はより極端な比率で成立している。
**結論**: **90-7-3則**を既定とする。
- 90%: ベース（無彩色または低彩度 C<0.03）
- 7%: セカンダリ（ベースの明度違い、ボーダー・カード背景）
- 3%: アクセント（CTA・リンク・強調のみ。使用箇所は1画面3箇所以内）
- アクセントの価値は希少性に比例する。増やすほど視線誘導力が下がる（後述のフォンレストルフ効果）

## 3. コントラスト — 定量基準

**提示**: WCAG 2.x のコントラスト比 4.5:1 を守ればよい。
**反証**: WCAG 2.xの式はダーク背景で誤差が大きい（暗背景の白文字を過大評価、細字を考慮しない）。APCA（WCAG 3草案）はフォントサイズ・ウェイト連動で人間の知覚に近い。ただしAPCAは未正式。
**結論**: 二重基準で運用する。
- 法的・実務ライン: WCAG 2.x — 本文 4.5:1 以上、大文字(24px+) 3:1 以上、UI部品 3:1 以上
- 品質ライン: APCA Lc 75+（本文16px）、Lc 60+（見出し）を目安
- ダークテーマでは純白 `#FFF` を避け `L≈0.93` に落とす（ハレーション防止）。純黒 `#000` 背景も避け `L≈0.13-0.18`（OLED滲みと目の疲労）

## 4. 色相調和 — 角度の数学

| 手法 | H角度差 | 用途 |
|---|---|---|
| モノクロマティック | 0°（L,Cのみ変化） | ミニマル・高級系。失敗が最も少ない |
| アナロガス | ±30° | 穏やかなブランド感 |
| 補色 | 180° | 最大の緊張感。アクセント専用 |
| スプリット補色 | 180°±30° | 補色より扱いやすい緊張感 |
| トライアド | 120°間隔 | 多色が必要なダッシュボード等 |

**反証と限界**: 角度理論は「破綻しない」保証であり「優れる」保証ではない。ギャラリー掲載サイトの多くはモノクロ＋単一アクセント（実質2色）。色数と洗練度は反比例する傾向。
**結論**: 迷ったら「無彩色ベース＋アクセント1色」。3色以上使う場合のみ角度理論を適用。

## 5. 色の意味論（言語的分解）

- 黒/ダークネイビー: 高級・技術・権威。「cool系」の既定値
- 白/オフホワイト: 誠実・医療・ミニマル。日本のコーポレートの既定値
- 彩度が下がるほど「大人・高価」、上がるほど「若年・安価・エネルギー」と知覚される（価格知覚研究と整合）
- 青は業種を問わず信頼のシグナル（世界の企業ロゴで最多）。ただし食品では食欲減退
- 文化差: 日本では赤=祝義もあるが、UI文脈では赤=エラーが国際的に固定。流用しない

## 6. 実装規則

1. 色は必ずセマンティックトークン化する（`--color-bg-primary`, `--color-accent`）。生HEXをコンポーネントに書かない
2. ホバー/アクティブはOKLCHのLを ±0.05〜0.08 動かして機械的に生成する
3. 影は黒の透過ではなく、背景色の色相を保った暗色を使うと濁らない（`oklch(from var(--bg) calc(l - 0.1) c h / 0.4)` 的発想）
4. グラデーションは同一色相内（H差 ≤ 30°）でLを動かす。色相をまたぐ場合はOKLCH補間（sRGB補間は中間が濁る）
