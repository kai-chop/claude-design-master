# レイアウト・余白 — 数学的分解と結論

## 1. 空間の単位系

**提示**: 余白は8ptグリッド（8の倍数）で統一する。
**反証**: 8pt単独では小さいUI（4px）と大きなセクション余白（96-160px）の両端で解像度が足りない。また8の倍数の等差列は大スケールで差が知覚できない（craft上、112と120の差は見えない）。
**結論**: **4px基底の準等比スケール**を使う。
```
4, 8, 12, 16, 24, 32, 48, 64, 96, 128, 160
（比率 ≈ 1.5前後の粗い等比。中間値を勝手に作らない）
```
- コンポーネント内: 4-16 / コンポーネント間: 24-48 / セクション間: 96-160
- **近接の法則の数値化**: 「関係が近い要素の距離 < 関係が遠い要素の距離」を常に1段以上のスケール差で保証する（例: カード内余白16なら、カード間は24以上）

## 2. 余白量と知覚品質

**提示**: 余白は多いほど高級に見える。
**反証**: 無条件ではない。余白が多い＝情報密度が低い＝1画面の伝達量が減る。ECや管理画面では余白過多は「不便」。また余白だけ増やしても要素の整列が乱れていれば安っぽい。
**結論**: 余白量はジャンル変数として扱う。
- ラグジュアリー/ブランド: セクション余白 120-200px、コンテンツ幅 640-800px
- コーポレート: 80-120px、幅 1080-1200px
- EC/メディア/管理画面: 48-80px、密度優先
- 共通則: **余白の階層 > 余白の総量**。整列（アライメント）が守られていることが前提条件

## 3. グリッド

- 基本: 12カラム、ガター24-32px、最大幅 1200-1440px
- 「cool系」で頻出する応用形:
  - **ブロークングリッド**: 要素をグリッド線から意図的に1-2カラムはみ出させ、画像とテキストを重ねる。ただし基底グリッドが存在するから「破れ」が知覚される。グリッドなしのランダム配置とは別物
  - **スプリットスクリーン**: 50:50 または 黄金比分割(38:62)で左右分割。片側固定＋片側スクロールが定番
  - **非対称余白**: 左余白 ≫ 右余白 で緊張感を作る（日本語横書きは左起点なので左に空間を置くと視線が流れる）

## 4. 比率の数学

**提示**: 黄金比 1:1.618 を使えば美しくなる。
**反証**: 黄金比の美的優位性は実証研究で再現性が低い（Fechner以降の追試は概ね否定的）。実務での価値は「美の魔法」ではなく「恣意性の排除＝一貫した比率系を持つこと」にある。
**結論**: 比率は何でもよいが**1つに固定して全域に適用する**ことに価値がある。
- 実用比率: 16:9（メディア）、4:3（カード画像）、3:4（ポートレート）、1:1（プロフィール）
- 画像アスペクト比をページ内で3種以下に制限する

## 5. 視覚的重量とバランス

- 要素の視覚的重量 ≈ 面積 × コントラスト × 彩度。数式ではなく発想として、大きい薄い要素と小さい濃い要素は釣り合う
- 左右非対称レイアウトは、この釣り合いで安定させる（例: 左に巨大な低コントラスト見出し、右に小さな高彩度CTA）
- **光学的整列 > 数学的整列**: 三角形アイコンや丸括弧は幾何中心と視覚中心がずれる。最終判断は目視。再生ボタンの三角は数px右にずらす

## 6. レスポンシブ規則

1. ブレークポイントはデバイスではなくコンテンツで決める（行長35字を超えたら段組みを検討、など）。実用既定: 640 / 1024 / 1440
2. セクション余白・見出しサイズは `clamp()` で流動化: `font-size: clamp(2rem, 1rem + 4vw, 4.5rem)`
3. モバイルでは余白スケールを1段下げる（96→64）が、コンポーネント内余白は維持する（タップ領域を守る）
4. タップターゲット最小 44×44px（Fitts's lawの実務下限）
