# 視覚階層・視線誘導 — 認知科学的分解と結論

## 1. 視線パターン

**提示**: 視線はF型（左上→右へ走査→下へ）またはZ型に流れるので、それに沿って配置する。
**反証**: F/Z型は「デザインが視線を制御していないとき」のデフォルト挙動（NN/gのアイトラッキング研究の本来の含意は「Fパターンは斜め読みされている失敗状態」）。強い視覚階層があれば視線はパターンを無視して重量順に跳ぶ。
**結論**: F/Zは下敷き、階層は上書き。
- テキスト主体ページ: F型を前提に、左端に情報のフック（見出し・箇条書き）を揃える
- LP/ヒーロー: Z型の4点（左上ロゴ→右上ナビ/CTA→左下コピー→右下CTA）を骨格にする
- 視線を跳ばせたい場所には「重量の山」（サイズ×コントラスト×彩度）を意図的に1つだけ作る

## 2. 誘導の手段（強い順）

1. **動き**: 周辺視野は動きに最も敏感。だからこそ1画面1モーションに制限（複数動くと全部が背景になる）
2. **人の顔・視線**: 人物写真の視線方向をユーザーは追う。モデルの視線をCTAへ向ける
3. **コントラスト・色の孤立**（フォンレストルフ効果）: 同質な集合の中の異質1点は必ず見られる。アクセント色を絞る理由はこれ
4. **サイズ**
5. **矢印・線・方向性のある形**: 明示的だが野暮になりやすい。斜め線やグラデーションの方向でも代替可
6. **余白による孤立**: 最も上品な強調。ラグジュアリー系の主武器

## 3. ゲシュタルト原則の実装対応

| 原則 | 実装ルール |
|---|---|
| 近接 | 関連要素の距離 < 非関連要素の距離（スケール1段差以上） |
| 類同 | 同機能=同スタイル。リンクの色・下線を全ページで統一 |
| 囲み/共通領域 | カード化は近接より強い。ただしカードのネストは2段まで |
| 連続 | 整列線を守る。1pxのズレは無意識に「安っぽさ」として知覚される |
| 閉合 | 画面端で画像を意図的に切ると「続きがある」＝スクロール誘発 |
| 図と地 | 主役以外の彩度・明度差を下げ、地に沈める |

## 4. 認知負荷の法則

- **ヒックの法則** `T = b·log₂(n+1)`: 選択肢が増えるほど決定時間は対数増加。ナビは7項目以下、CTAは1画面に主1つ（副CTAは視覚重量を明確に下げる）
- **フィッツの法則** `T = a + b·log₂(2D/W)`: ターゲットは大きく・近く。主CTAは親指圏（モバイル下部）、関連アクションは発生源の近くに置く
- **ミラーの法則（7±2 → 実務では4±1）**: 情報は3-4個のチャンクに分割。特徴紹介が6個あるなら3×2ではなく「3つの柱＋詳細」に再構造化を検討
- **系列位置効果**: 最初と最後が記憶に残る。ナビの両端・ページの最上部/最下部に最重要項目

## 5. 階層設計の手順（アルゴリズム）

1. ページの要素に重要度スコアを付ける（1=最重要CTA 〜 5=法的表記）
2. スコア1は1画面に1つしか存在できない。複数あれば要件定義に戻る
3. 各スコア差につき視覚変数を2つ以上変える（サイズ/ウェイト/色L/余白/彩度）
4. **squintテスト**: 目を細める（またはブラー10pxをかける）。スコア1→2→3の順に知覚できなければ階層は失敗
5. 5秒テスト: 初見5秒で「何のサイト・何をすべきか」が答えられるか

## 6. 視認性の下限（アクセシビリティ＝人間工学の底面）

- コントラスト: 01-color.md の基準に従う
- 色だけで意味を伝えない（エラーは色＋アイコン＋文言）
- フォーカスリングを消さない。`:focus-visible` でスタイルする
- 情報を伝えるインタラクションはホバー専用にしない（タッチデバイスに存在しない）
