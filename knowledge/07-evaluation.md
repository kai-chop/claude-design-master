# 評価・反証プロセス — 完成判定の手続き

「一般人に優れていると評価される」を検証可能にするためのチェック体系。
実装後、公開前に必ず全項目を実行する。

## 1. 定量チェック（機械的に検証可能・不合格は即修正）

| # | 項目 | 基準 |
|---|---|---|
| Q1 | 本文コントラスト | WCAG 4.5:1 以上（大文字3:1） |
| Q2 | 本文サイズ | 16px以上、行長 日本語35字/欧文66字前後 |
| Q3 | 行間 | 本文1.6-1.9 / 見出し1.1-1.3 |
| Q4 | 余白スケール逸脱 | スケール外の値（例: 13px, 27px）が0件 |
| Q5 | 使用色数 | セマンティックトークン以外の生色が0件 |
| Q6 | アクセント色の出現箇所 | 1画面3箇所以下 |
| Q7 | フォント | 2ファミリー・3ウェイト以下 |
| Q8 | タップ領域 | 44×44px以上 |
| Q9 | モーション | transform/opacityのみ、reduced-motion対応 |
| Q10 | 整列 | グリッド線から根拠なくズレた要素0件（ブロークングリッドは「根拠あるズレ」） |

## 2. 知覚チェック（人間工学的・squint系）

- **ブラーテスト**: スクショに10pxブラー。視覚階層スコア1→2→3の順で知覚できるか
- **5秒テスト**: 初見5秒で「何のサイトか」「何をすべきか」が言えるか
- **グレースケールテスト**: 白黒化しても階層・状態（エラー等）が判別できるか
- **380px幅テスト**: 最小モバイルで横スクロール・文字潰れがないか
- **1画面1主役テスト**: 各ビューポートで最も目立つ要素を指させるか。2つ以上あれば失敗

## 3. 反証ラウンド（設計の自己批判。最低1周する）

完成案に対し、以下の反証質問を立て、耐えられなければ修正して再提示する:

1. **逆ジャンル検査**: このデザインは意図と逆のジャンル（cool狙い→安っぽい/子供っぽい）に見える要素を1つでも含むか。（典型犯: 彩度過多、既定easing、整列ズレ、低品質画像、角丸の不統一）
2. **削除検査**: 各装飾要素を1つずつ「消したら悪くなるか」自問。悪くならないなら消す。ギャラリー級デザインとの差は多くの場合「足りない」ではなく「多い」
3. **模倣検査**: 参考サイトの表層（色・動き）だけ真似て、構造（階層・余白の論理）を落としていないか
4. **初見者検査**: 制作文脈を知らない人が誤解する導線・文言はないか。内輪の論理で組んでいないか
5. **負荷検査**: LCP 2.5s以内か。美しくても遅いサイトは一般人評価では「悪いサイト」

## 4. 総合スコアリング（雛形）

各10点、70点未満はリリース不可・反証ラウンドへ戻る:
配色 / タイポ / 余白・整列 / 階層・誘導 / モーション / レスポンシブ / パフォーマンス / 第一印象（形容詞3つが意図と一致するか）

## 5. このナレッジ自体の限界（メタ反証）

- 本体系は「破綻の防止」と「掲載水準の再現」を保証するが、「独創性」は保証しない。独創は原則を1つだけ意図的に破ることから生まれる（2つ以上破ると単なる破綻）
- 数値は2020年代のディスプレイ・ブラウザ環境前提。基準値は定期的に実サイト観察で校正すること
- ギャラリー掲載≠事業成果。CV目的のサイトではA/Bテストの実測が本書に優先する
